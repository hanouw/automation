# 블로그 자동화 프로젝트 통합 진행 이력 (2026-04-06 ~ 04-08)

## 1. 프로젝트 개요
- **목표**: Gemini AI를 활용한 고품질 블로그 콘텐츠(글+이미지) 자동 생성 및 주요 플랫폼(네이버, 티스토리, 구글) 자동 업로드 시스템 구축.
- **주력 플랫폼**: 티스토리 (성공), 네이버 (보류), 구글 블로거 (예정).

## 2. 주요 기술적 성과

### [티스토리 자동 업로드 (`tistory_uploader.py`) - 완벽 성공]
- **브라우저 제어**: Playwright의 `launch_persistent_context`를 사용하여 카카오/티스토리 로그인 세션을 영구 유지.
- **HTML 모드 자동화**: 
  - 에디터 상단 모드 선택 버튼(`#editor-mode-layer-btn-open`)을 제어.
  - 키보드 방향키(`ArrowDown` 3회)와 `Enter`를 조합하여 'HTML 모드'로의 전환 성공.
- **본문 입력 기술**: `CodeMirror` 엔진 내부의 `.CodeMirror-line`을 타겟팅하여 클립보드 복사-붙여넣기(`Ctrl+V`) 방식으로 대량의 HTML 코드를 오차 없이 주입.
- **멀티 이미지 자동 삽입 (NEW)**:
  - **Placeholder 시스템**: `generator.py`가 본문 중간에 `<!-- IMAGE_PLACEHOLDER_N -->` 주석을 생성하도록 고도화.
  - **선 업로드 후 치환**: `images_generated` 폴더의 이미지를 먼저 업로드한 뒤, 추출된 실제 이미지 태그를 본문의 Placeholder 위치와 자동 치환하여 삽입하는 로직 구현.
  - **파일 가로채기**: `expect_file_chooser()`를 사용하여 브라우저 파일 선택 시스템 자동화.

### [Gemini 웹 기반 이미지 생성 (`gemini_web_generator.py`) - 혁신적 성공]
- **API 한계 극복**: 구글 API의 할당량(429 Error) 문제를 해결하기 위해 Gemini 웹 사이트를 직접 제어하는 방식 도입.
- **고퀄리티 템플릿 시스템**:
  - '이미지 만들기' 도구 내의 다양한 템플릿(단색, 활주로, 시네마틱 등)을 **랜덤 선택**하여 시각적 다양성 확보.
- **자동 다운로드 및 파일 관리**:
  - `data-test-id="download-generated-image-button"` 속성을 정밀 타겟팅하여 고화질 원본 이미지 다운로드 성공.
  - `expect_download(timeout=120000)`를 통해 대용량 이미지 다운로드가 완료될 때까지 안정적으로 대기.

### [텍스트 생성 엔진 (`generator.py`)]
- **모델**: `gemini-3-flash-preview` (안정성 및 할당량 최적화).
- **멀티 스타일**: 네이버(친근), 티스토리(전문적/HTML), 구글(SEO) 맞춤형 글 동시 생성.
- **이미지 프롬프트 연동**: 글 내용에 최적화된 영문 이미지 생성 프롬프트를 함께 추출하여 연동성 강화.

## 3. 환경 및 보안 설정
- **라이브러리**: `playwright`, `pyperclip`, `Pillow`, `python-dotenv`, `google-generativeai`.
- **보안**: `.gitignore`를 통해 민감한 세션 데이터 및 생성물 폴더 자동 제외 처리.
