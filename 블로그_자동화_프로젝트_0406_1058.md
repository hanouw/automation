# 블로그 자동화 프로젝트 진행 이력 (2026-04-06)

## 1. 프로젝트 개요
- **목표**: Gemini AI를 활용한 블로그 포스팅 생성 및 네이버/티스토리/구글 블로그 자동 업로드 시스템 구축.
- **언어**: Python (가상환경 `venv` 사용)
- **주요 도구**: `google-generativeai` (Gemini API), `playwright` (브라우저 자동화), `python-dotenv` (환경 변수 관리).

## 2. 주요 작업 내역

### [환경 설정 및 기초 구현]
- **가상환경 설정**: 전역 패키지 충돌 방지를 위해 `venv` 환경 구축 및 `pip install -r requirements.txt` 완료.
- **콘텐츠 생성기 (`generator.py`)**: 
  - Gemini 2.0 Flash 모델을 사용하여 네이버(친근), 티스토리(전문적), 구글(SEO) 맞춤형 글 생성.
  - 결과물을 `text_generated` 폴더 내에 JSON 및 TXT 형식으로 자동 분류 저장 (파일명: `blogtxt_MMDD_HHMM`).

### [네이버 업로드 자동화 고도화 (`naver_uploader.py`)]
- **외부 기술 분석 반영**: 
  - GitHub(`icodetree/naver-auto-write`) 분석을 통해 네이버 스마트에디터의 `mainFrame` iframe 구조 파악 및 적용.
- **세션 유지 및 로그인 로직**:
  - `launch_persistent_context`를 활용하여 `naver_user_data` 폴더에 세션 저장. 
  - 로그인 미완료 시 무한 대기(`timeout=0`) 및 리다이렉트 자동 감지.
- **스마트에디터 제어 최적화**:
  - **iframe 전환**: `mainFrame`으로의 정확한 전환을 통해 에디터 내부 요소 접근성 확보.
  - **방해 요소 자동 제거**: '도움말' 패널 및 '작성 중 글 불러오기' 팝업 자동 닫기 구현.
  - **입력 안정성**: 제목 영역 다중 셀렉터 지원 및 한 글자씩 입력(`type`)하는 인간적인 타이핑 방식 도입.
  - **본문 구조화**: 줄바꿈(`\n`)을 고려한 본문 자동 입력 시스템 구축.
- **발행 프로세스 자동화**:
  - 발행 버튼의 다중 셀렉터(클래스명, 텍스트 조합) 지원 및 최종 확인 팝업 클릭 로직 완성.
  - 발행 실패 시 백업 로직(Enter 키 입력 등) 추가로 성공률 극대화.

## 3. 향후 계획
- **이미지 생성 연동**: 생성된 글에 어울리는 이미지를 AI로 생성하여 자동 삽입하는 기능 검토.
- **타 플랫폼 확장**: 티스토리 및 구글 블로거(Blogger) API/자동화 업로드 모듈 추가 구현.
- **스케줄링**: Windows 작업 스케줄러를 통한 매일 정해진 시간 자동 실행 테스트.
