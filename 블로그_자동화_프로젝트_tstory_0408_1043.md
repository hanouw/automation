# 블로그 자동화 프로젝트 진행 이력 (2026-04-06 ~ 04-07)

## 1. 프로젝트 현황
- **주력 플랫폼**: 티스토리 (Playwright 브라우저 자동화 방식)
- **콘텐츠 생성**: Gemini API (`generator.py`)
- **이미지 생성**: Gemini 웹 자동 제어 (`gemini_web_generator.py`)

## 2. 주요 업데이트 및 성과

### [티스토리 자동화 최종 최적화]
- **HTML 모드 전환 성공**: 에디터 상단 모드 선택 드롭다운(`#editor-mode-layer-btn-open`)을 제어하여 'HTML 모드'로 자동 전환 구현.
- **CodeMirror 입력 해결**: 티스토리 특유의 코드 에디터(`.CodeMirror-line`)를 타겟팅하여 본문 HTML 내용을 클립보드 복사-붙여넣기로 주입 성공.
- **키보드 시뮬레이션**: 팝업 취소(ArrowRight + Enter) 및 모드 선택(ArrowDown 3회 + Enter)을 통해 물리적 조작 수준의 안정성 확보.

### [이미지 생성 전략 혁신 (`gemini_web_generator.py`)]
- **API 한계 돌파**: 구글 Gemini API의 이미지 생성 할당량 이슈(429 Error)를 해결하기 위해 **Gemini 웹 사이트 직접 제어** 방식으로 전환.
- **템플릿 시스템 도입**: 
  - Gemini 웹의 '이미지 만들기' 도구 진입 자동화.
  - 제공되는 고퀄리티 템플릿(단색, 활주로, 시네마틱 등) 중 하나를 **랜덤 선택**하여 생성물의 예술적 퀄리티 대폭 향상.
- **자동 다운로드 구현**: 
  - 사용자 제공 HTML 구조(`mat-icon[data-mat-icon-name="download"]`)를 반영하여 고화질 이미지 자동 다운로드 로직 완성.
  - `expect_download` 기능을 통해 생성된 이미지를 `images_generated` 폴더에 체계적으로 저장.

### [시스템 보안 및 환경 관리]
- **Git 관리 최적화**: `.gitignore`를 업데이트하여 `naver_user_data`, `tistory_user_data`, `google_user_data` 등 민감한 세션 폴더를 제외 처리.
- **라이브러리 보강**: 클립보드 이미지 처리를 위해 `Pillow` 라이브러리 추가 도입.
