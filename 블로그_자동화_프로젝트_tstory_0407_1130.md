# 티스토리 블로그 자동화 프로젝트 진행 이력 (2026-04-06)

- **기존 상황**: 네이버 블로그 자동화 시도 중 에디터의 복잡성 및 보안 이슈로 인해 티스토리로 주력 플랫폼 변경.
- **전환 결정**: 티스토리 공식 API 종료에 따라 **Playwright(브라우저 제어)** 방식을 고도화하여 구현 성공.

## 2. 주요 성과 및 업데이트 내역

### [티스토리 업로드 자동화 성공 (2026-04-06 20:30)]
- **첫 실사용 성공**: `tistory_uploader.py`를 통해 실제 티스토리 블로그에 글 작성 및 발행 전 과정이 **완벽하게 자동화**됨을 확인.
- **간편 로그인 돌파**: 카카오 계정 선택 화면을 `.wrap_profile` 클릭 및 키보드 조작 조합으로 안정적으로 통과.

### [고품질 서식 적용 및 HTML 모드 안정화]
- **HTML 모드 전환 (Success)**: 에디터 상단 모드 선택 버튼(`#editor-mode-layer-btn-open`) 클릭 및 방향키 조작을 통해 'HTML 모드' 전환 완벽 성공.
- **방해 요소 제거**: "이전 글 이어쓰기" 팝업 발생 시 방향키 및 Enter 조합으로 자동 취소 처리.
- **CodeMirror 입력 기술 해결**: `.CodeMirror-line` 요소를 타겟팅하여 HTML 모드에서의 본문 주입 문제 해결.

### [이미지 생성 기능 도입 시도 (`image_generator.py`)]
- **시도 내용**: 구글 Gemini API의 차세대 이미지 모델군(코드네임: **Nano Banana**)을 활용한 자동 이미지 생성 시도.
- **현황 및 이슈**:
  - `gemini-2.5-flash-image`, `gemini-3.1-flash-image-preview` 등 다수의 모델로 테스트 수행.
  - **429 Quota Exceeded (limit: 0)** 오류 지속 발생: 현재 사용자의 구글 API 프로젝트에서 이미지 생성 권한이 미활성화되어 있거나 지역/티어 제한으로 인해 사용이 불가능한 상태 확인.
- **결론**: 구글 API를 통한 이미지 생성은 일시 보류하며, 나노바나나(외부 서비스) 또는 OpenAI DALL-E 3 등 대체 도구 도입 검토 중.

## 3. 기술적 특징
- **키보드 시뮬레이션**: 요소 클릭이 불안정한 드롭다운이나 팝업을 방향키와 엔터로 제어하여 안정성 극대화.
- **클립보드 연동**: `pyperclip`을 사용하여 대량의 텍스트를 한 번에 주입함으로써 타이핑 오류 및 속도 문제 해결.

## 4. 향후 계획
- **이미지 생성 도구 확정**: 외부 API 연동을 통한 이미지 자동화 재시도.
