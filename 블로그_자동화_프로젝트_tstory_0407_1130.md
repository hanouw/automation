# 티스토리 블로그 자동화 프로젝트 진행 이력 (2026-04-06)

## 1. 프로젝트 배경 및 전환
- **기존 상황**: 네이버 블로그 자동화 시도 중 에디터의 복잡성 및 보안 이슈로 인해 티스토리로 주력 플랫폼 변경.
- **전환 결정**: 티스토리 공식 API 종료에 따라 **Playwright(브라우저 제어)** 방식을 고도화하여 구현 성공.

## 2. 주요 성과 및 업데이트 내역

### [티스토리 업로드 자동화 성공 (2026-04-06 20:30)]
- **첫 실사용 성공**: `tistory_uploader.py`를 통해 실제 티스토리 블로그에 글 작성 및 발행 전 과정이 **완벽하게 자동화**됨을 확인.
- **간편 로그인 돌파**: 카카오 계정 선택 화면을 `.wrap_profile` 클릭 및 Tab 키 조합으로 안정적으로 통과.

### [고품질 서식 적용 (HTML 모드 전환)]
- **이슈**: 생성된 글에 포함된 HTML 태그(<h2>, <ul> 등)가 기본 모드에서 텍스트로 노출되는 문제 발생.
- **해결 (방법 A 적용)**: `tistory_uploader.py`에서 본문을 입력하기 전 에디터를 **'HTML 모드'**로 자동 전환하도록 로직 추가.
- **결과**: AI가 생성한 풍부한 HTML 서식이 실제 블로그 포스팅에서 완벽하게 렌더링되어 전문적인 레이아웃 확보.

### [텍스트 생성 엔진 복구 (`generator.py`)]
- **업데이트**: 고품질 서식 확보를 위해 다시 HTML 태그를 포함하여 글을 생성하도록 프롬프트 복구 및 최적화 완료.

### [발행 및 공개 설정 자동화]
- **공개 설정**: 발행 레이어에서 `id="open20"`(공개) 라디오 버튼을 자동 클릭하고 `id="publish-btn"`으로 최종 발행하도록 보강 완료.

- **이미지 생성 연동**: 글 내용에 맞는 이미지를 AI로 생성하여 본문에 삽입하는 로직 구현 검토.
- **완전 자동 스케줄링**: `generator` -> `uploader`를 하나로 묶어 매일 정해진 시간에 자동 포스팅하는 시스템 구축.
